First Spacings

When I started Arendelle I putted a special part to one of my first Arendelle programs called 'First Spacings' a part of the program that sets where our sketch will start. Somehow First Spacings made it to the first of every Arendelle program and it developed to an Arendelle habit and soon a part of Arendelle programmers sub culture.

First Spacings of other books are usually "Who this book is for" or "Where to start", "How to read this book". And so now I have to write some of this for my book!

Requirements

You are not required to know anything! When I started Arendelle I wanted to design a clean language and simplicity was the most important factor and the result was something so simple like no other language.

Simplicity of Arendelle made it one of the best languages for learning programming principles. Also Arendelle is a language for visual graphics which is an artist's job. So because we are targeting artists as the main Arendelle users and also we want to use Arendelle as a way to attract kids and people with no past experience to programming I will write this book to teach programming principles by playing with Arendelle and visual graphics.

So either if you're an experienced developer who can code a whole operating system in binary with a pen and some papers without even debugging it once or if you're not a developer and you are just curios about how programming works or if you have a cool idea and you want to make it working this is your book all you need to use this book is some love and time.

Should I read the whole book?

When it comes to programming books most of us only read one or two paragraphs of only one or two chapters that we need. We (In this case I) read this way because most of the times we know languages and we only need one or two special techniques or codes and it's always easier to find the answer on the internet. If you're one of this people this book might is in some way not good for you because Arendelle is a different language than others, When you're reading a Java book you always know half of the book because you know C family or when you're reading a Python one you know all of it because you know Ruby and reading the book is something you don't need.

Most of the programming languages are this way. The act of creating a language is mostly to add features or shaping a language for a specific domain so they mostly use another language's grammar. That's why in C, C++, C#, Java and... you use one same for control. But Arendelle is not like another language. It's grammar, the way it works, it's commands, variable system... Everything in Arendelle is designed from scratch that's why even if you know all programming languages out there you still can't read Arendelle codes without reading something about it.

That's why I'm asking you to read "Basics of Arendelle" chapter, and the "Loop" chapter. But here's a good news: There is a "Cheat Sheet" chapter for you at the end of this book so you can use it as a quick reference.
